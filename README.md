# HSK Portal — Учебная практика

## Статус проекта
✅ **Бэкенд**: Django + MySQL — работает  
✅ **Авторизация**: вход по email/паролю с JWT  
✅ **Роли**: администратор (`admin`), сотрудник (`employee`)  
✅ **Задачи**: CRUD, фильтрация по статусу, назначение  
✅ **Новости**: публикация для всех, удаление — только админу  
✅ **Админка пользователей**: создание, блокировка, изменение ролей  
✅ **Фронтенд**: React SPA с маршрутизацией и локальным сохранением  
✅ **Админка**: управление пользователями (только админ)  
✅ **Интерфейс**: аккуратный, единый стиль

## Как запустить бэкенд
1. Перейдите в папку `backend`
2. Активируйте виртуальное окружение: `venv\Scripts\activate`
3. Установите зависимости: `pip install -r requirements.txt`
4. Запустите сервер: `python manage.py runserver`
Адрес: http://localhost:8000
Админка: http://localhost:8000/admin/

## Фронтенд (React)
1. cd frontend
2. npm install
3. npm start
Адрес: http://localhost:3000
Автоматически открывает /login.

## Функционал
Задачи:
Админ видит все задачи, может создавать.
Сотрудник видит только свои.
Фильтрация: «все», «выполнено», «не выполнено».
Удаление и смена статуса — только админом или создателем.
Новости:
Все авторизованные могут публиковать.
Только админ может удалять.
Журналирование:
Фиксация кто и когда создал задачу/новость, кто поменял статус.

## Примечание 

В процессе разработки я столкнулся с технической проблемой — циклическими импортами и конфликтами в истории Git,
которые мешали запуску сервера. Чтобы не терять время и не рисковать стабильностью проекта, я решил восстановить рабочее состояние,
сохранив все функциональные изменения. Всё, что было сделано ранее (авторизация, задачи, новости),
было перенесено в текущую версию вручную, с проверкой на работоспособность. История коммитов была очищена для упрощения отладки,
но код и функциональность полностью сохранены.
